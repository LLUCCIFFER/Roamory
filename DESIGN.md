> 版本：v1.1 参考图对齐版

> 设计方法：基于 `frontend-design` 技能进行产品化视觉收敛

## 1. 设计结论

参考图的核心气质是“蓝白水彩旅行梦境 + 个人日记主页 + 目的地轮播”。Roamory 的界面应从原先偏理性的“地图手账”进一步调整为：

```text
会规划路线的云端旅行日记
```

它应该像一个柔和、轻盈、可收藏的虚拟旅行空间，而不是普通 AI 工具、攻略平台或效率型行程表。

界面第一屏要像参考图一样直接给出“Choose a Destination”的行动感，同时保持产品功能清晰：

1. 选择或输入目的地。
2. 浏览推荐目的地/季节灵感。
3. 生成可编辑行程。
4. 保存为我的旅行日记和足迹。

## 2. 参考图视觉分析

### 2.1 可借鉴元素

| 参考图元素 | Roamory 转化 |
|---|---|
| 蓝白水彩天空背景 | App 首屏、空状态、分享页背景。 |
| 远山、云朵、海边栈桥 | 目的地选择、行程封面、推荐轮播素材。 |
| 白色蕾丝边分隔 | 页面段落分隔、海报装饰、生成中进度边框。 |
| 胶囊导航条 | 顶部/底部主导航、分段控件、Day tabs。 |
| 半透明浅蓝玻璃卡片 | 创建表单、个人日记卡、行程摘要卡。 |
| 缩略图轮播 | 推荐目的地、行程封面、海报模板。 |
| 白色手绘花朵和蝴蝶 | 低密度背景装饰，只用于情绪页。 |
| 手写英文 quote | 分享海报和旅行记忆页的装饰性文案。 |

### 2.2 需要克制的地方

- 参考图整体偏装饰型网页，真实产品页不能牺牲可读性。
- 手写字体只能用于短 quote，不用于导航、按钮、表单或核心数据。
- 白色低对比文字和按钮需要提升对比度，保证移动端可读。
- 蕾丝、花朵、蝴蝶只用在首屏、空状态、分享海报，不进入密集编辑页。
- 导航胶囊可以大圆角，功能卡片仍以 8px 圆角为主，避免所有元素都像装饰贴纸。

## 4. 视觉方向

### 4.1 设计命题

用“水彩天空 + 目的地轮播 + 日记玻璃卡”的语言表达 AI 旅行记忆。

产品不是把行程表做得更冷静，而是让用户感觉每一次计划都像在打开一页轻盈的旅行日记。

### 4.2 视觉结构

首屏建议接近参考图的结构：

```text
┌──────────────────────────────┐
│ Choose a Destination          │
│ [ 规划 ] [ 足迹 ] [ 我的 ]     │
│                              │
│ ┌──────────────────────────┐ │
│ │ 大目的地封面 / 轮播       │ │
│ └──────────────────────────┘ │
│      <  SUMMER  >             │
│  [thumb] [thumb] [thumb]      │
│                              │
│ ┌──────────────────────────┐ │
│ │ 输入目的地 / 生成按钮     │ │
│ └──────────────────────────┘ │
└──────────────────────────────┘
```

### 4.3 记忆点

标志性元素：

```text
Cloud Diary Carousel
```

它是参考图中“目的地轮播 + 季节切换 + 缩略图”的产品化版本：

- 首页展示 3-5 个目的地封面。
- 中间有季节/氛围切换，例如 Summer、Sunset、Weekend、Rainy Day。
- 缩略图下方进入创建旅行。
- 生成后的行程会变成一张 “Travel Diary Card”。

原先的 `Memory Atlas Strip` 调整为二级元素，用在行程结果、足迹和海报中，负责表达路线和旅行记录；首页主记忆点改为 `Cloud Diary Carousel`。

## 5. 色彩系统

参考图是高明度蓝白体系，但产品需要更强的文本对比。色板保留清透感，同时加入可读的蓝灰和少量粉色。

| Token | Hex | 用途 |
|---|---|---|
| `sky-wash` | `#BFE8FA` | 大面积水彩天空背景。 |
| `cloud-white` | `#F8FCFF` | 卡片、输入框、浅色面板。 |
| `mist-blue` | `#D9F1FB` | 半透明玻璃面板、hover 背景。 |
| `sea-glass` | `#7DC9E6` | 主按钮、选中态、进度点亮。 |
| `lake-ink` | `#4F7484` | 标题、导航、主图标。 |
| `deep-ink` | `#243D4A` | 正文与关键数据，保证对比。 |
| `mountain-blue` | `#7FA7BD` | 次级按钮、地图/远山线条。 |
| `lace-white` | `#FFFFFF` | 蕾丝边、分隔和高光。 |
| `petal-pink` | `#F5B8CB` | 花朵、收藏、温柔强调。 |
| `sand-note` | `#F6EEDB` | 旅行票据、提示卡片。 |

使用规则：

- 主背景用 `sky-wash` 与水彩纹理，不使用纯渐变当作完整视觉。
- 正文必须使用 `deep-ink`，不要用过浅蓝灰。
- 主 CTA 使用 `sea-glass` 背景和 `deep-ink` 或白色文字，按实际对比测试调整。
- `petal-pink` 只用于收藏、花朵、引用点缀，不作为大面积主色。
- 密集工具页减少装饰，使用 `cloud-white` 和 `mist-blue` 保持可读。

## 6. 字体系统

参考图标题是圆润英文展示字，正文较轻，底部 quote 使用手写体。Roamory 可这样落地：

| 场景 | 字体 |
|---|---|
| 英文展示标题 | `Quicksand`, `Nunito`, system-ui |
| 中文标题 | `Noto Sans SC`, `LXGW WenKai`, system-ui |
| 中文正文/UI | `Inter`, `Noto Sans SC`, system-ui |
| 数据/时间/坐标 | `JetBrains Mono`, `SFMono-Regular`, monospace |
| 装饰 quote | `Playfair Display Italic`, `Cormorant Garamond Italic`, serif |

字号建议：

| Token | Size | 用途 |
|---|---:|---|
| `display` | 30-34 | 首页 “Choose a Destination”。 |
| `title` | 22-24 | 页面标题。 |
| `section` | 17-18 | 分区标题。 |
| `body` | 15-16 | 正文。 |
| `caption` | 12-13 | 辅助信息、标签。 |
| `data` | 13-14 | 预算、通勤、评分。 |

排版规则：

- 不使用负字距。
- 展示标题可以轻微圆润、宽松，但中文功能标题要可读。
- 手写/斜体只用于海报 quote、日记短句，不用于操作控件。

## 7. 背景与装饰

### 7.1 水彩天空背景

推荐使用位图或 CSS layered background：

- 第一层：浅蓝底色。
- 第二层：云雾水彩纹理。
- 第三层：低透明远山轮廓。
- 第四层：少量白色手绘花朵/蝴蝶。

要求：

- 背景不能影响文字对比。
- 移动端背景要固定视觉重心，避免花纹压住内容。
- 不用离散发光球、渐变球或 bokeh 装饰。

### 7.2 蕾丝边

参考图里的白色蕾丝边很关键，可以作为 Roamory 的分隔语言。

使用位置：

- 首页大轮播和内容区之间。
- 生成中进度条底部。
- 分享海报边框。
- 空状态底部。

不使用位置：

- 表单内部。
- 地图编辑页。
- 隐私/错误提示。
- 密集列表之间。

### 7.3 玻璃日记卡

面板风格：

```css
background: rgba(248, 252, 255, 0.72);
backdrop-filter: blur(18px);
border: 1px solid rgba(255, 255, 255, 0.72);
box-shadow: 0 18px 48px rgba(61, 118, 145, 0.18);
```

规则：

- 大型情绪面板可以使用玻璃拟态。
- 功能卡片保持清晰边界和稳定尺寸。
- 文本区不要透明过度，保证可读。

## 8. 布局原则

### 8.1 移动端首屏

移动端应保留参考图的纵向梦幻结构，但把任务入口前置：

```text
┌─────────────────────┐
│ Choose a Destination │
├─────────────────────┤
│ 胶囊导航             │
├─────────────────────┤
│ 大目的地轮播         │
│ SUMMER / WEEKEND     │
│ 缩略图               │
├─────────────────────┤
│ 目的地输入 + CTA     │
├─────────────────────┤
│ 最近行程 / 足迹摘要  │
└─────────────────────┘
```

### 8.2 桌面/Web 验证版

桌面端可以更贴近参考图的三段式：

```text
┌───────────────────────────────────────┐
│ Choose a Destination                  │
│ [About Me] [Home] [Virtual Space]     │
├───────────────────────────────────────┤
│ 目的地大轮播 + 季节切换               │
├───────────────────┬───────────────────┤
│ Travel Diary Card │  Quote / Poster   │
└───────────────────┴───────────────────┘
```

产品化命名建议：

- `About Me` -> `我的`
- `Home` -> `规划`
- `Virtual Space` -> `足迹空间`

### 8.3 卡片使用

卡片只用于具体对象：

- 目的地封面。
- Travel Diary Card。
- POI 卡片。
- Day 卡片。
- 城市足迹。
- 权限说明。

圆角规则：

- 胶囊导航和分段控件可以 full radius。
- 功能卡片默认 8px radius。
- 图片封面和海报预览可以 10-12px radius，以贴合参考图，但不滥用。

## 9. 核心页面设计

### 9.1 规划首页

目标：让用户像选一张旅行明信片一样开始规划。

组件：

- 英文/中文混合展示标题：`Choose a Destination` / `选择目的地`。
- 胶囊导航：规划、足迹空间、我的日记。
- 大目的地轮播。
- 季节/氛围切换：Summer、Weekend、Sunset、Rainy Day。
- 缩略图条。
- 目的地输入框。
- 主 CTA：生成我的行程。
- 最近行程或旅行日记卡。

设计要点：

- 首屏保留一张真实或生成的目的地图片，不用纯插画替代。
- 输入和 CTA 不要藏在轮播之后，移动端首屏必须能看到操作入口。
- 胶囊导航高度固定，避免文字撑开。

### 9.2 创建旅行页

控件：

| 字段 | 控件 |
|---|---|
| 目的地 | 搜索输入 |
| 出发城市 | 定位 + 搜索 |
| 日期 | 日期选择器 |
| 天数 | Stepper |
| 预算 | 金额输入 + 预算等级 |
| 兴趣 | 多选标签 |
| 同行人 | Segmented control |
| 节奏 | Segmented control |
| 交通方式 | 多选按钮 |
| 补充需求 | 多行文本 |

设计要点：

- 表单承载在玻璃日记卡里，但字段区域用 `cloud-white` 提升可读性。
- 长表单分成“去哪”“怎么走”“喜欢什么”三段。
- 底部固定主 CTA。
- 背景装饰透明度降低，避免干扰输入。

### 9.3 AI 生成中页

生成中页使用“云层逐步散开”的概念，步骤仍然清楚：

```text
理解偏好 -> 搜索地点 -> 计算通勤 -> 安排节奏 -> 生成日记卡
```

状态文案：

- 正在理解你的旅行偏好
- 正在寻找适合你的地点
- 正在计算点位之间的通勤时间
- 正在安排每日节奏
- 正在生成旅行日记卡

动效：

- 目的地卡片轻微浮入。
- 进度点像缩略图逐个点亮。
- reduced motion 下只保留淡入和进度文本。

### 9.4 行程结果页

结构：

```text
┌─────────────────────┐
│ Travel Diary Card    │
│ 标题 / 评分 / 城市   │
├─────────────────────┤
│ Day 胶囊 tabs        │
├─────────────────────┤
│ 时间轴 + POI cards   │
├─────────────────────┤
│ 保存 / 编辑 / 分享   │
└─────────────────────┘
```

每个 POI 卡片：

- 编号。
- 名称。
- 类型。
- 推荐理由。
- 停留时间。
- 到下一站通勤时间。
- 操作：锁定、替换、删除。

风格：

- 标题区域可以梦幻，时间轴区域必须清晰。
- POI 卡片使用功能卡样式，不要过度透明。
- Day tabs 使用参考图胶囊导航语言。

### 9.5 地图/路线编辑页

移动端：

- 上方地图/路线缩略区域。
- 下方可拖拽时间轴。
- 底部保存按钮。

桌面端：

- 左侧时间轴。
- 右侧地图。

设计限制：

- 编辑页减少花朵、蝴蝶、蕾丝。
- 保留浅蓝背景和胶囊 tabs 即可。
- 拖拽、替换、删除状态要比装饰更重要。

### 9.6 足迹页

足迹页可以更接近参考图的 `Virtual Space`：

- 背景保持水彩天空。
- 中心是一张个人旅行日记卡。
- 下方是点亮城市缩略卡。
- 城市详情像一页日记，而不是数据报表。

MVP 模块：

- 已点亮城市数。
- 最近一次旅行。
- 城市列表。
- 手动点亮入口。
- 从相册找回旅行入口，P1 灰度。

### 9.7 分享海报页

海报应直接继承参考图语言：

- 蓝白水彩背景。
- 白色蕾丝边。
- 目的地图片。
- 简短旅行 quote。
- 日记卡式行程摘要。

必须展示隐私检查：

```text
即将分享：
- 行程城市
- 每日路线
- 景点名称
- 通勤时间
- 预算：不包含
- 精确位置：不包含
```

## 10. 组件规范

### 10.1 Capsule Nav

参考图核心组件。

规则：

- 高度 36-44px。
- full radius。
- 外层浅色描边。
- 选中项使用 `cloud-white` 填充和轻微阴影。
- 文案短，不超过 5 个中文字符或 14 个英文字符。

### 10.2 Button

| 类型 | 样式 |
|---|---|
| Primary | `sea-glass` 背景，白字或 `deep-ink`。 |
| Secondary | `cloud-white` 背景，`lake-ink` 描边。 |
| Ghost | 透明背景，`lake-ink` 文本。 |
| Danger | `petal-pink` 浅底，`deep-ink` 文本。 |
| Icon | 40x40，使用 lucide 图标。 |

按钮文案使用动作：

- 生成我的行程
- 保存行程
- 替换地点
- 点亮城市
- 生成海报

### 10.3 Destination Carousel

元素：

- 大图。
- 左右切换按钮。
- 季节/氛围标签。
- 缩略图。
- 当前项标题。

要求：

- 大图比例固定，建议 `16:9` 或 `5:3`。
- 缩略图数量 3-5 个。
- 切换按钮用图标，不使用长文字。
- 图片必须有真实旅行感，避免暗、糊、过裁切的素材。

### 10.4 Travel Diary Card

用于个人旅行摘要和足迹。

元素：

- 封面/头像/目的地图。
- 标题。
- 日期。
- 城市。
- 简短摘要。
- 统计信息。
- CTA。

样式：

- 半透明玻璃面板。
- 内部文本区域增加浅白遮罩。
- 功能卡片圆角 8px。

### 10.5 Chips

用于兴趣、节奏、交通方式、季节。

状态：

- Default：`cloud-white` 背景。
- Selected：`mist-blue` 背景，`lake-ink` 文本。
- Highlight：`petal-pink` 或 `sand-note` 浅底。
- Disabled：低对比描边。

### 10.6 Timeline

时间轴是功能核心，装饰必须克制。

元素：

- 时间。
- POI 编号。
- 停留时长。
- 到下一站通勤。
- 风险提示。

状态：

- locked。
- hidden。
- pending route。
- route failed。

### 10.7 Privacy Notice

隐私不是警告恐吓，而是清晰控制。

写法：

```text
默认不会公开你的预算、酒店位置和精确轨迹。
分享前你可以逐项确认。
```

## 11. 动效原则

使用少量有意义动效：

- 首页目的地轮播轻微滑动。
- 云层背景非常慢地漂移，或完全静态。
- 生成中缩略图逐个点亮。
- 保存行程后日记卡盖章式反馈。
- 分享海报生成完成后预览轻微浮入。

限制：

- 尊重 `prefers-reduced-motion`。
- 不使用粒子雨、发光球、漂浮装饰堆叠。
- 地图和时间轴交互优先流畅。

---

## 12. 响应式规则

| 宽度 | 布局 |
|---|---|
| `< 768px` | 单列移动端，胶囊导航和底部 Tab。 |
| `768-1199px` | 内容居中，轮播 + 表单上下排列。 |
| `>= 1200px` | 上方大轮播，下方日记卡/海报/足迹双栏。 |

固定格式组件必须有稳定尺寸：

- 胶囊导航高度固定。
- 大轮播使用固定 `aspect-ratio`。
- POI 卡片最小高度固定。
- 海报模板使用固定比例。
- 缩略图尺寸固定，不能因图片比例跳动。

## 13. 文案风格

文案要轻盈，但操作必须明确。

推荐：

- Choose a Destination
- 选择目的地
- 生成我的行程
- 保存为旅行日记
- 点亮这座城市
- 今天太满了，帮我放松一点
- 通勤时间待确认
- 分享前检查公开内容

避免：

- 开启梦幻旅程
- 探索无限可能
- AI 赋能你的诗与远方
- 一键解锁人生地图

装饰 quote 可以更柔和，例如：

```text
Slow down, breathe deeply, and collect the little joys.
```

## 14. 设计自检

每轮界面开发后检查：

- 是否明显继承参考图的蓝白水彩、胶囊导航、玻璃卡片和轮播结构。
- 首屏是否能直接开始选择目的地。
- 装饰是否只出现在合适页面，没有干扰表单和编辑。
- 核心文字在移动端是否足够清晰。
- 图片、轮播、海报是否有固定比例，不造成布局跳动。
- 按钮文字是否表达明确动作。
- 隐私相关信息是否足够清楚。
- 是否避免把整个产品做成纯装饰网页，仍能完成真实规划流程。
